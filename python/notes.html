🐍 Python Notes – Conceptual View

1️⃣ Introduction to Python

What is Python?

Python is a high-level, interpreted, general-purpose programming language designed with the philosophy:

“Code should be readable and simple.”

Unlike low-level languages such as C and Assembly, Python hides hardware complexity and allows the programmer to focus on logic and problem-solving.

Why Python Was Created

Guido van Rossum created Python to:

  • Reduce code complexity
  • Improve readability
  • Allow rapid development
  • Make programming accessible to beginners

Python follows The Zen of Python:

  • Simple is better than complex
  • Readability counts
  • Explicit is better than implicit

3️⃣ Python Execution Model (Internal Working)

How Python Code Runs

  • Source code (.py) is written
  • Python interpreter converts it into bytecode
  • Bytecode runs on Python Virtual Machine (PVM)

👉 This is why Python is slower than C, but faster to develop.

4️⃣ Variables & Memory Management

What is a Variable?

A variable is a reference to a memory object, not a container.

x = 10

Here:

  • 10 is an object
  • x points to that object

Key Concepts

  • Python uses automatic memory allocation
  • Garbage collection removes unused objects

5️⃣ Data Types – Conceptual Understanding

Immutable Types

  • int, float, str, tuple
  • Cannot be changed after creation
x = 5
x = x + 1 # new object created

Mutable Types

  • list, set, dict
  • Can be modified in place
lst = [1, 2]
lst.append(3)

6️⃣ Control Flow – Logical Decision Making

Conditional Statements

Conditional statements are used to control program behavior based on conditions.

if condition:
    statement
Python uses indentation instead of braces, which enforces clean and readable logic.

7️⃣ Loops – Iterative Thinking

for Loop

Used when the number of iterations is known.

for i in range(5):
    print(i)

while Loop

Used when iterations depend on a condition.

while condition:
    action

Loop Control Statements

  • break → terminate loop
  • continue → skip current iteration
  • pass → placeholder (does nothing)

8️⃣ Functions – Code Abstraction

Why Functions Exist

  • Avoid repetition
  • Improve readability
  • Support modular programming
  • Enable easy debugging
def add(a, b):
    return a + b

Types of Functions

  • Built-in functions
  • User-defined functions
  • Lambda (anonymous) functions

9️⃣ Python Collections (Deep Explanation)

List

  • Ordered
  • Mutable
  • Allows duplicate elements
Used when: Data changes frequently

Tuple

  • Ordered
  • Immutable
Used when: Data integrity is required

Set

  • Unordered
  • No duplicate elements
Used when: Unique elements are needed

Dictionary

  • Stores data in key-value pairs
  • Provides fast data lookup
Used in:
  • Databases
  • JSON data
  • APIs

🔟 Exception Handling – Defensive Programming

What is an Exception?

An exception is an unexpected event that disrupts the normal flow of program execution.

try:
    risky code
except:
    handle error

Purpose of Exception Handling

  • Prevent program crash
  • Improve reliability
  • Provide user-friendly error messages

1️⃣1️⃣ File Handling – Persistent Storage

Why File Handling?

Variables store data temporarily, but files allow data to be stored permanently.

File Modes

  • r → read
  • w → write
  • a → append

1️⃣2️⃣ Object-Oriented Programming (OOP)

Core OOP Concepts

Class

A class is a blueprint for creating objects.

Object

An object is an instance of a class.

class Student:
    def __init__(self, name):
        self.name = name

OOP Pillars (Very Important)

  • Encapsulation → Binding data and methods together
  • Inheritance → Reusing existing class features
  • Polymorphism → One interface, many forms
  • Abstraction → Hiding internal details
